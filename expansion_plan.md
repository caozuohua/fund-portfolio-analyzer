## 1. 当前架构分析

### 数据流

```
akshare（基金净值接口）
    ↓
analyze.py（拉取数据 + 计算持仓分析）
    ↓
report.md（生成 Markdown 周报）
    ↓
ai_analyze.py（调用 Gemini/Groq AI 生成分析建议）
    ↓
email（发送周报邮件）
```

### 优势

- **全自动化**：GitHub Actions 定时触发，零人工干预
- **零服务器成本**：利用 GitHub 免费 Actions 额度
- **数据可追溯**：每次运行产出 `report_YYYYMMDD.md`，历史报告完整保留
- **AI 增强**：接入 Gemini/Groq，自动生成专业分析建议

### 局限

- **仅日频数据**：akshare `fund_open_fund_daily_em` 接口只返回最近 2 天数据，无法获取历史净值序列
- **无历史净值序列**：无法计算真实的技术指标（RSI、MA、波动率等）
- **无回测能力**：无法验证过去 AI 建议的实际效果
- **无操作记录**：无法追踪持仓变更历史和累计收益
- **单一通知渠道**：仅支持邮件，缺少即时通知选项

## 2. 短期拓展（1-2 周可完成）

### 2.1 持仓变更追踪系统 🥇

**目标**：记录每次买入/卖出操作，追踪操作后收益，让每次决策都有据可查。

**实现方案**：

1. 新建 `config/holdings_log.json`，结构如下：

```json
{
  "operations": [
    {
      "date": "2026-06-15",
      "action": "买入",
      "fund_code": "008774",
      "fund_name": "招商鑫福中短债债券A",
      "amount": 50000.00,
      "nav": 1.195,
      "shares": 41841.00,
      "reason": "固收类配置不足，加仓短债提升稳定性",
      "source": "AI建议"
    }
  ]
}
```

2. 在 `analyze.py` 中增加 `log_operation()` 函数，支持 CLI 调用：

```bash
python analyze.py --log-op --action 买入 --code 008774 --amount 50000 --reason "加仓固收"
```

3. 周报新增板块：
   - **本周操作**：列出本周所有买入/卖出记录
   - **操作后收益**：对比操作时净值与当前净值，计算实际收益/亏损
   - **累计操作收益**：所有历史操作的总收益汇总

### 2.2 回测验证引擎 🥉

**目标**：验证过去 AI 建议的实际效果，形成"建议→验证→改进"的闭环。

**实现方案**：

1. 新建 `scripts/backtest.py`
2. 输入：某次 AI 建议（如"卖出 A 基金 X 元，买入 B 基金"）
3. 逻辑：
   - 获取建议日期至今的净值序列
   - 计算"若执行建议"的收益率
   - 计算"若未操作"的收益率
   - 对比两者差异
4. 输出示例：

```
📊 历史建议复盘
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
建议日期: 2026-05-15
建议内容: 卖出 017704（货币）5万元 → 买入 008774（短债）

若执行建议: +1.23%（+615元）
若维持原状: +0.41%（+205元）
建议贡献:   +0.82%（+410元）✅ 建议有效
```

5. 周报新增"历史建议复盘"板块，自动复盘最近 4 周的建议

### 2.3 净值序列补全 🥈

**目标**：获取 60 日历史净值序列，计算真实技术指标。

**实现方案**：

1. 使用 `akshare.fund_etf_hist_em()` 或 `akshare.fund_hist_min_em()` 补取 60 日净值
2. 新建 `scripts/fetch_nav_history.py`，每日运行并追加到 `data/nav_history/{fund_code}.csv`
3. 计算技术指标：
   - MA5 / MA20 / MA60（移动平均线）
   - RSI(14)（相对强弱指标）
   - 20日波动率（年化）
   - 最大回撤（近 60 日）
4. 周报新增"技术指标"板块，标注超买/超卖信号

## 3. 中期拓展（1-2 个月）

### 3.1 全市场分析

**目标**：将分析范围从持仓基金扩展到全球主要资产类别，提供宏观视角。

**实现方案**：

1. 新增数据源：

| 资产类别 | 标的 | akshare 接口 |
|---------|------|-------------|
| A股大盘 | 沪深300 (000300) | `index_zh_a_hist` |
| A股中小盘 | 中证500 (000905) | `index_zh_a_hist` |
| 美股 | SPY (标普500 ETF) | `fund_etf_hist_em` |
| 黄金 | GLD (黄金ETF) | `fund_etf_hist_em` |
| 债券 | TLT (长期国债ETF) | `fund_etf_hist_em` |

2. 周报新增"全球市场概览"板块：
   - 各资产近 1 周 / 1 月 / 3 月涨跌幅
   - 相关性热力图（持仓基金 vs 大类资产）
3. 市场极端情况自动预警：
   - VIX > 30 → 高风险预警
   - 单日跌幅 > 3% → 市场异动提醒
   - 沪深300 跌破 MA60 → 趋势转弱信号

### 3.2 智能调仓建议

**目标**：基于现代投资组合理论，给出量化调仓建议。

**实现方案**：

1. 实现均值方差优化（Markowitz）或风险平价模型
2. 输入：
   - 当前持仓权重
   - 各资产历史波动率（60日年化）
   - 资产间相关性矩阵
3. 输出：
   - 最优配置权重 vs 当前权重
   - 具体调整建议（卖出 X → 买入 Y）
4. 保守型约束条件：
   - 单一基金占比 ≤ 15%
   - 权益类总占比 ≤ 30%
   - 货币类 ≥ 20%（流动性底线）
   - 每次调仓幅度 ≤ 5%（避免过度交易）

### 3.3 多账户支持

**目标**：支持多个券商/平台的持仓汇总管理。

**实现方案**：

1. 重构 `holdings.json` 为多账户结构：

```json
{
  "accounts": [
    {
      "name": "华泰证券",
      "type": "券商",
      "holdings": [...]
    },
    {
      "name": "支付宝",
      "type": "第三方平台",
      "holdings": [...]
    }
  ]
}
```

2. 合并计算总资产、总配置比例
3. 周报按账户分别展示 + 合并汇总

## 4. 长期愿景（3-6 个月）

### 4.1 个人投资操作系统

**目标**：从"周报分析工具"升级为"投资决策支持系统"。

**核心功能**：

- **持仓管理**：实时持仓概览 + 多账户汇总
- **操作记录**：完整操作日志 + 操作理由归档
- **收益归因**：拆解收益来源（资产配置 vs 个基选择 vs 择时）
- **风险评估**：VaR、最大回撤、夏普比率等风险指标
- **智能建议**：AI + 量化模型双重驱动的操作建议

**目标**：让你每次操作都有数据支撑，告别情绪化决策。

### 4.2 社区化

**目标**：做成可复用的 skill/template，降低其他用户的使用门槛。

**实现方案**：

1. 抽象为通用模板，持仓配置外部化
2. 增加交互式配置向导：

```bash
python setup.py
# → 选择风险偏好（保守/平衡/进取）
# → 输入持仓基金代码
# → 设置目标配置比例
# → 选择 AI 提供商（Gemini/Groq/OpenAI）
# → 选择通知渠道（Email/Telegram/Discord）
```

3. 完善 README 和文档，提供一键部署指南
4. 发布为 Hermes Agent skill，支持 `hermes install fund-portfolio-analyzer`

## 5. 技术债务和优化

### 5.1 数据持久化

- 历史净值存到 `data/nav_history/` 目录
- 每次运行自动提交到 git，形成完整数据时间线
- 数据格式：CSV（`date, nav, accumulated_nav`）

### 5.2 测试覆盖

- 为 `analyze.py` 增加单元测试（pytest）
- 覆盖场景：数据拉取失败、持仓为空、极端净值波动
- GitHub Actions 中增加 CI 检查

### 5.3 错误恢复

- 数据拉取失败时的降级策略：
  - 重试 3 次（间隔 30s）
  - 使用上次缓存数据（`output/latest_data.json`）
  - 邮件通知用户数据异常
- AI 分析失败时跳过，仅发送基础数据报告

### 5.4 通知渠道扩展

- 增加 Telegram Bot 通知（即时性优于邮件）
- 增加 Discord Webhook 通知
- 用户可配置通知级别（仅异常 / 每次报告）

## 6. 优先级建议

按 ROI（投入产出比）排序：

| 优先级 | 功能 | 理由 | 预估工时 |
|--------|------|------|----------|
| 🥇 P0 | 持仓变更追踪系统 | 直接提升决策质量，是后续所有功能的基础 | 3-5 天 |
| 🥈 P1 | 净值序列补全 + 真实技术指标 | 解锁技术分析能力，为回测和调仓提供数据基础 | 2-3 天 |
| 🥉 P2 | 全市场分析 | 提供宏观视角，帮助判断大方向 | 3-4 天 |
| P3 | 回测验证引擎 | 形成建议闭环，持续改进 AI 建议质量 | 4-6 天 |
| P4 | 智能调仓模型 | 量化建议比纯 AI 建议更可靠 | 5-8 天 |
| P5 | 多账户支持 | 功能增强，但非核心需求 | 2-3 天 |

### 推荐执行路径

```
Week 1-2:  持仓变更追踪 → 净值序列补全（夯实数据基础）
Week 3-4:  全市场分析 → 回测验证引擎（扩展分析能力）
Month 2:   智能调仓模型 → 多账户支持（量化升级）
Month 3+:  投资操作系统 → 社区化（产品化）
```
